hexo + gitpages 搭建博客

发表于 2016-04-03 | 更新于 2017-04-07 | 分类于 配置 | | 阅读次数 |

本篇文章主要记录用hexo搭建博客，并部署在github上的大概过程。

搭建博客并部署

下载并安装node.js

安装node.js

下载并安装 git

安装git
进行简要的配置，详细信息参照廖雪峰git教程

在 github 上建立一个 repo

repo名字为github名字.github.io，比如 AlexVon.github.io

在本地创建一个git库，与repo重名

git init AlexVon.github.io

安装 hexo

使用 npm 安装 hexo

npm install hexo --save
#如果命令无法运行，可以尝试更换taobao的npm源
npm install -g cnpm --registry=https://registry.npm.taobao.org

初始化 hexo 文件夹

在 git 本地库中

hexo init
npm install
#新建完成后，指定文件夹的目录如下
.
├── _config.yml  #全局配置文件
├── package.json
├── scaffolds   # 模版文件,新建文章时会根据 scaffold里对应md生成文件
├── scripts
├── public  # hexo g 后产生的文件夹，后面部署时上传的也是这个文件夹
├── source
|      └── _posts # 发布的文章
└── themes  #主题文件

安装 hexo插件

# 在部署的时候使用
npm install hexo-deployer-git --save

查看效果

hexo g # 生成public静态文件
hexo s # 开启服务器

如果没有错误，可以在本地打开浏览器，打开网址 http://localhost:4000/

部署在github

修改文件 AlexVon.github.io_config.yml,找到 deploy 这一项，这是我的配置

deploy:
  type: git
  # 也可以使用ssh
  repo: https://github.com/VonAlex/VonAlex.github.io.git
  branch: master # 分支一定要相同

然后回到 AlexVon.github.io 文件夹

hexo d # 部署

没有发生错误，然后就可以正常访问http://vonalex.github.io/了。
执行该命令后，会在 hexo 文件下多出一个 .deploy_git 的文件夹。

主题配置

本教程采用的是 next 主题

下载主题

在 hexo 的文件下使用 git 命令来获得主题文件。

git clone https://github.com/iissnan/hexo-theme-next themes/next

这时在 themes 文件夹下就多出了一个next的主题文件夹.

配置

添加分类页面

hexo new page categories

这时候，会在 hexo 的 source 文件夹下多出来一个 categories 文件夹，修改该文件下的 index.md 文件如下：

---
title: categories
date: 2016-04-02 21:38:55
type: "categories"
---

添加标签页面

hexo new page tags

修改 source\tags\index.md 文件如下：

---
title: tags
date: 2016-04-02 21:40:22
type: "tags"
---

同理可以使用 hexo new page 页面名来添加其他的页面，然后在主题配置文件中加上图标对应关系，若使用的是中文主题，那还要在主题文件夹下的 language 文件下的中文文件中进行设置，不然会出现英文。
menu 的根目录/是 hexo 文件夹下的 source 文件夹。

在需要变迁的文章中加入 tgs 属性:
tags: [spark,sparkSQL] # 添加spark和sparkSQL两个标签

导航栏新加页面

大体方法跟上面添加 tags 类似，这里举一个具体的栗子，因为有一些在线文档需要经常查看，所以我要增加一个在线文档的页面：
首先， hexo new page doconline，doconline为新页面，这时会在 source 目录下出来一个 doconline 的文件夹，里面只有一个index.md 的文件。因为这个页面不需要评论，所以添加一行代码 comments: false。
然后，配置文件 _config.yml，在 menu 这个项目中添加 doconline: /doconline，就是让导航栏能够找到这个页面。
然后，在 menu_icons 这个项中为这个新加的页面增加一个想要的图标，这个上面有讲过。
最后，在 themes/next/languages/zh-Hans.yml 中找到 menu 这个项，配置要让新的页面显示什么中文名字。
同理，在英文主题文件中修改。
一定要注意名字要对应， 为放置出错，一律用相同的小写字母就行了！

其他一些配置，比如设置头像、设置评论、添加百度统计等等都可以看这里next主题的配置，本文不再啰嗦。

使用不蒜子脚本

sitemap

使用 npm 安装 hexo-generator-sitemap 插件，安装方法跟上面一样。
将 hexo 站点配置文件中的 url 这一项的值修改成你的博客地址，
pugins 这一项的值修改为 hexo-generator-sitemap，然后在后面添加

sitemap:
    path: sitemap.xml

这一项其实不加也可以，当使用 hexo g 生成文件时，后在 source 文件夹下产生一个 sitemap.xml 文件，默认路径就是它。
hexo-generator-sitemap 插件是针对谷歌的，若要被百度收录，还需要安装 hexo-generator-baidu-sitemap 插件，设置同上。
至于如何提交 sitemap.xml 文件，见参考9

其他

图标问题

next主题的图标采用的是fontawesome免费图标，比如在页面中加入这条语句就可以引用 github-alt 这个图标。

<i class="fa fa-github-alt"></i>

在主题配置文件 _config.yml，设置 Social Links Icons 时用到了图标，比如：

social:
  GitHub: https://github.com/VonAlex

social_icons:
  enable: true
  GitHub: github-alt

如上设置后，就为 https://github.com/VonAlex这个链接设置了一个 github-alt 图标。

要注意 social 和 social_icons 中 icon 名字要对应，github-alt 是到http://fontawesome.io/icons/网站查找到的图标名称，在页面代码中会对它进行引用。menu 和 menu_icons 的对应也是这样。
另外，这些图标在 md 文件中也能用

md文件中插图问题

插图方法
  1. 可以用 md 中的语法![]()，图片大小不可控。
  2. 使用 标签，因为 html 的标签可以无缝使用。
图片位置
  1. 可以单独建立一个文件夹，专门存放图片。比如 hexo 下，可以在 source 文件夹下建立 images 文件夹，用于存放图片。
  2. 使用图床，有一些免费图床, 可以搜索到，还可以使用七牛、又拍等的云存储服务。

几个文件的作用

next\layout\_partials\footer.swig 控制页脚显示
next\layout\_layout.swig 控制整个页面的显示
next\layout_macro\sidebar.swig 控制侧边的显示
next\layout\page.swig 控制page页面
next\layout\index.swig 控制首页的显示
themes\next\layout_partials\header.swig 用于生成当行栏中的各个页面的内容和图标，以及链接
source : 站点存放资源的位置,该目录下的 markdown 和 html 文件均会被hexo处理
.deploy：执行 hexo d 命令部署到 GitHub 上的内容目录

代码中的 theme.* 将读取 next_config.yml 文件中的相应字段，比如 theme.since 那么读取的就是 since 这个配置项中的数据。

常用的 hexo 命令
注意：这些命令要在 hexo 文件下使用

npm install hexo -g #安装  
npm update hexo -g #升级  
hexo init # 初始化
hexo cl == hexo clean 清理之前生成的public文件夹
hexo new page 页面名 #新建页面
# 新建文章（文章名不用加引号），在站点下source\_posts下生成"文章名.md"
hexo n  文章名 == hexo new 文章名
hexo g == hexo g # 生成要发布博客的所有静态资源
hexo s == hexo server #  开启本地服务器可以进行预览
hexo d  == hexo deploy # 部署
hexo deploy -g # 生成并部署
hexo server -g # 生成并预览
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP
# hexo
